Internet and Website Development

Introduction to the World Wide Web (WWW)

The World Wide Web is a system of interlinked public web pages that are accessible through the internet. It contains a vast variety of documents, accessible by the user, that are all linked by hyperlinks. Hyperlinks highlight a word or phrase that redirects you to another document that provides further information about the word or phrase you originally clicked on. The WWW uses web servers, which are computer programs that store and transmit documents to other computers when requested. Clients are programs that request the documents from the server. Web browsers are software applications that allow the user to view and access the documents stored within the web, for example, Google Chrome, Microsoft Edge, Safari, and Firefox.

How is the World Wide Web different from the internet?

The Web

The web is a system created by Tim Berners-Lee in 1989, using HyperText Transfer Protocol (HTTP), the coding language of web pages, standardising the communication between clients and servers. It serves as the world’s main service to retrieve information from the internet.

The Internet

The internet is a global system of networks, connecting computers and servers globally, being first developed in the mid 20th century, and became publicly available in the early 1900s. The internet is used for virtually any purpose that involves information and can be accessed by anyone that has a connection to a network. It has many ways of sharing information, such as the World Wide Web, email, File Transfer Protocol (FTP), and instant messaging services.

Altogether, the WWW and the internet are two different technologies that vastly help internet users in accessing information. The WWW is just one part of the internet that works with it to provide information, entertainment and many other services to the public. The internet links every computer, and transports the information, and from this, the web stores it and allows you to access and use the information on web browsers.

What is an intranet?

An intranet is a software that is separate from but similar to the internet but still uses an internalised private network that is used for securely containing and sharing information. It relates to the internet because of how they both store and share information and give access to users to view it. They connect users and computers within a network, and also both use standard protocols, transmission control protocol/internet protocol (TCP/IP), and both use web browsers.

How does it relate to the internet?

Although the intranet is a lot more private, being restricted to a select amount of users, the internet is accessible by practically anyone globally. An intranet is useful for organisations as their information can be stored privately and securely from public eyes and provides only employees within the organisation with access to the information, and is owned by said organisation. Because of this, users within an intranet enjoy the benefits of a private network, i.e. the information can only be accessed by authorised computers within the network, a limited amount of users, limited amount of information for specific users, its own IP address for the network and users in the network, and an intranet is much safer and secure than the internet.

What is an internet protocol?

The Internet Protocol (IP), is a network layer communications protocol for routing and addressing data across networks. IP information is attached to packets, data which is divided into smaller pieces, and this information helps get the data within the packets, to its right destination on the internet. Every domain on the internet has an IP address, a unique identifier assigned to the device, which is where packets would be directed to. In order for data to be transported to where it is needed.

Compare HTTP, TCP/IP and FTP.

Hypertext Transfer Protocol (HTTP)

HTTP, the standard application layer protocol, is the protocol that stands as the foundation for the entire web. It helps HTTP clients, which are web browsers, send file requests to web servers in order to exchange files to open and load web pages. Once the request is accepted by the server, the user will be granted access to the information, which will be displayed on the user’s screen for them to view. HTTP relies on the TCP standard, as a TCP/IP connection is required before a file request can take place.

Transmission Control Protocol (TCP)

TCP, is a transport layer network protocol that allows programs and devices to communicate with each other over a network. It establishes a connection between the source and destination before transmission and makes sure it remains live until the communication begins, and then breaks down large amounts of data into smaller packets, ensuring the reliable transmission of data.

TCP/IP

TCP frequently works together with another protocol, IP, to transfer data between destinations, as a result, making a TCP/IP connection. IP provides the address of the destination, where the data is to be sent, and TCP is then responsible for the transmission of the data to that destination, once the IP address has been obtained and identified.

File Transfer Protocol (FTP

FTP, is an application layer communications protocol that is responsible for the transferring of files between a client and server. It uses a TCP/IP connection to then transmit files between computers. In an FTP transaction, both computers, usually called the ‘local host’ and the ‘remote host’ must be connected to a network, have FTP software installed, be able to transfer files via FTP, and the server must be able to run FTP services.

Protocol Purpose
HyperText Transfer Protocol (HTTP) Standard application layer protocol used to send file requests to web servers to open and load web pages.
Transmission Control Protocol (TCP) Transport layer network protocol that allows communication between programs and ensures the reliable transmission of data.
Internet Protocol (IP) Network layer communications protocol responsible for routing and addressing data so they can get to the correct destination.
File Transfer Protocol (FTP) Application layer communications protocol that allows for the transfer of files between a client and server, via a TCP/IP connection.
